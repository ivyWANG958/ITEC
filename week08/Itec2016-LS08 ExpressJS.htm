Today, the HTML form development will be discussed. A simple web application will also be developed using the Express.js framework to handle the form submission.

Express.js is a web application framework that runs on top of Node.js, which is a server-side JavaScript runtime environment. This means that the server application could also be developed with JavaScript.

HTML forms are used to collect user input and contain form elements like input elements, checkboxes, radio buttons, submit buttons, etc.

Form Elements

The <form> tag is used to create an HTML form for user input. The <form> element can contain one or more of the following form elements:

  • <input>
  • <textarea>
  • <button>
  • <select>
  • <option>
  • <optgroup>
  • <fieldset>
  • <label>

Fieldset

The <fieldset> tag is used to group related elements in a form. It draws a box around the related elements.

The <Input> Element

The <input> tag specifies an input field where the user can enter data. <input> elements are used within a <form> element to declare input controls that allow users to input data. An input field can vary in many ways, depending on the type attribute.
button date email image password reset tel url
checkbox datetime file month radio search text week
color datetime-local hidden number range submit time

Examples: (Radio buttons)

<div>
    <span>Gender:</span><br>
    <span>
          <input type="radio" value="Male">Male
          <input type="radio" value="Female">Female
    </span>
</div>

Place the above in the More form elements section of our form. Reload this page to see the changes.

The <Select> Element

The <select> element is used to create a drop-down list.

The <option> tags inside the <select> element define the available options in the list.

Let's add the following code to develop a pair of 2-level interdependent drop-down lists.

<div>
    <span>Select your favourite team:</span>
    <select onchange="teamSelected(this.value)">
        <option value="">Please select a team:</option>
        <option value="Avengers">Avengers</option>
        <option value="JLA">Justice League</option>
    </select>
</div>

<div>
    <span>Select your favourite hero:</span>
    <select id="hero" disabled>
    </select>
</div>

With onchange attribute, whenever there's change on the selected value, the teamSelected() function will be executed (to be provided later). And now, we've also disabled the second drop-down list, as its options will be dynamically created based on the selection on the first list.

Dynamic Options

At the very bottom of your code, just above the first <script>, let's add an area to develop our JavaScript code. Firstly, we develop two arrays to store the content of each team.

<script>
    var avengers = ["Captain America", "Iron Man", "Thor", "Hulk", "Black Widow", "Hawkeye"];
    var jla = ["Superman", "Batman", "Wonder Woman", "Flash", "Green Lantern", "Aquaman"];
</script>

Next, append the following function to the arrays. We'd like to know what team has been selected.

function teamSelected(teamValue) {

    alert(teamValue);

}

Here, we use an alert box to display the selected value, which was provided during the function call.

Next, we obtain a reference to the second drop-down list. The last line basically clears all the options in the second drop-down list.

var heroSelect = document.getElementById("hero");
heroSelect.options.length = 0;

Then, based on the option selected, we will fill the second drop-down list differently. Obviously, we need a conditional statement (if) here.

if (teamValue == "Avengers") {

    avengers.forEach(function (item) {
        var option = document.createElement("option");
        option.text = item;
        option.value = item;
        heroSelect.add(option);
    });

    heroSelect.disabled = false;

} else if (teamValue == "JLA") {

    jla.forEach(function (item) {
        var option = document.createElement("option");
        option.text = item;
        option.value = item;
        heroSelect.add(option);
    });

    heroSelect.disabled = false;

} else {

    heroSelect.disabled = true;

}

Here, we also enable the second drop-down list with heroSelect.disabled = false, when the user picks either Avengers or Justice League. Notice the use of if-then-else here.

TextArea

The <textarea> tag defines a multi-line text input control. A text area can hold an unlimited number of characters, and the text renders in a fixed-width font (usually Courier).

The size of a text area can be specified by the cols and rows attributes, or even better; through CSS's height and width properties.

<div>
    <span>Remarks:</span><br>
    <textarea cols=50 rows=12></textarea>
</div>

Input Validation

We can perform some basic client-side validations with HTML5.

Required fields

Validating required fields is as easy as adding the required attribute to each <input> tag that must be completed. For example, the name field now looks like this:

<div>
    <label for="name">Name:</label>
    <input type="text" id="name" required>
</div>

If we submit this form now without completing any of the required fields, the browser will notify us.

Validating emails and numbers

To make sure our user enters the right data in the email and number of tickets fields, we can use some of the new input types found in HTML5:

<input type="email" id="email" required>
...
<input type="number" id="numTickets" required>

Suppose we want to restrict the number of tickets each person could buy, we can achieve this by using the max and min attributes:

<input type="number" id="numTickets" min=1 max=4>

If the user enters anything less than 1 or greater than 4, they'll be prompted to enter a number in the permitted range.

Form Submission

When we click the Submit button, error would be displayed because we haven't defined the action to process this form. We haven't developed our server yet, thus let's use a free online service for this purpose. Add the following to the <form> opening tag:

action="https://www.httpbin.org/post"

HTTP Method - GET and POST

There are two different ways that a form can be submitted from your browser to the webserver. In HTML, this is how one would define the opening form tags for both submission methods: <form method="GET"> and <form method="POST">.

Example of a GET request

If a "GET" request is used, the form parameters are encoded in the URL in what is called a query string. The form parameters can be anything. Heres an example of the query string that would be generated if we were to use a "GET" request:

http://www.someemailprovider.com/?login=joe@email.com&password=xxyz

In the GET request above, you can see that the form parameters (login and password) are attached to the end of the URL itself. Note that defining a login form to use the GET request method as we did in this example is a very bad idea. This is because people logging in will see their passwords being displayed in the url and may be led to think that your site is not secure. One should almost always use a POST form whenever passwords are involved, for other reasons that are explained below.

A POST request, unlike a GET request, passes the form parameters in the body of the HTTP request, not in the URL. This happens behind the scenes.

Reference: http://www.programmerinterview.com/index.php/general-miscellaneous/html-get-vs-post/

Rule of Thumb:

  • Form contains password: Use POST
  • Form modifies the server-side database : Use POST
  • Form would not modify the server-side database: Use GET

As now we are adding content to the server-side database, we use POST, let's add the following to the <form> opening tag:

method="post"

Server-side Scripting with Express.js

Can we develop our server application to handle this form submission? We can develop a simple one quickly with the use of Express.js framework. This framework runs on top of a server-side JavaScript Runtime -- Node.js.

Node.js

Node.js has already been installed in our machines. To install it in your own machines, please visit the official website https://nodejs.org/en/download/ to download and install the latest LTS (long term support) version.

Express.js

One can directly develop a web server with Node.js, but it is definitely quite complicated as functionalities such as system and path handling, url resolution and parsing, file I/O access, etc ... are not included by default. To speed up the development, we can use the handy Express.js web application framework.

To install express.js, please bring up your terminal and run

su xdeveloper

sudo npm -g install express-generator 

exit

Both sudo and su are very important and mostly used commands in Mac/Linux. sudo allow users (sudoers) to run programs with the security privileges of another user, by default the superuser. This allows the express-generator to be successfully installed in the machine. However, your student user isn't a sudoer, thus we will have to use su to switch to another account, xdeveloper, which is a sudoer created by our technical team. The exit at the end will switch back to your original account.

Please note that you don't have to run the first line in your own Mac, and the sudo is also not needed for Windows system.

npm is the package manager of Node.js. It can install a number of useful packages and modules. Here, we installed the express-generator, which can generate an "express" project with a simple command:

express --view=ejs lab8

This line would generate a new folder lab8 with a skeleton of files automatically created inside (i.e., Scaffolding). ejs represents our chosen template engine, there're some other options like jade and handlebars.

Next, go inside this folder and install the required dependencies:

cd lab8

npm install

This simply fetches and installs all the required Node.js packages and modules, being listed in the package.json file. It takes a while.

Also copy our index.html to the public folder of this newly created project. Finally, we can start the server with 

npm start

By default, the server would run on port 3000, thus we can visit this website via http://localhost:3000/. You may have to turn off the proxy in case it can't be reached.

Now you should see the same form we developed before.

Echoing the From Inputs

To echo the incoming data like httpbin.org? It's fairly easy. Locate the file index.js inside folder routes, register the following route handler ...

/* Handle the Form */
router.post('/form', function(req, res, next) {
  
  res.json(req.body);
  
});

Whenever a client visits the path /form with the POST method, we will send back the request body back to the client.

Modify the action in our index.html:

action="/form"

Stop your server by pressing Ctrl + C. Restart it again and submit the form to our server.

If you find this JSON response not easy to read, add the following line to the file app.js, around line 16.

app.set('json spaces', 2);

This will "prettify" the JSON output.

Saving Data

We can connect our system to a number of database engines like MongoDB, MySQL or Oracle. But for simplicity, let's simply store the data in a local database file using Realm. The Realm database is a popular choice in implementing local database in mobile application development.

In the terminal, stop the server and issue the following command to add the Realm functionality in our system:

npm install realm --save

Then, near the beginning of index.js, add the following lines to load this Realm module and set up a database scheme to store the incoming data.

const Realm = require('realm');

const BookingSchema = {
  name: 'Booking',
  properties: {
    name: {type: "string", optional: true},
    email: {type: "string", optional: true},
    numTickets: {type: "int", optional: true},
    gender: {type: "string", optional: true},
    team: {type: "string", optional: true},
    hero: {type: "string", optional: true},
    remarks: {type: "string", optional: true}
  }
};

var bookingRealm = new Realm({
  path: 'booking.realm',
  schema: [BookingSchema]
});

In our route handler, add the following lines to store data in the server-side before we send response to the client.

req.body.numTickets = parseInt(req.body.numTickets) || undefined;

bookingRealm.write(function () {
    bookingRealm.create('Booking', req.body);
});

Line 1 converts the input into an integer. If fails, we simply use undefined, meaning the value is not available. Lines 3-5 save the entire form data to this Realm database.

In your LaunchPad, locate the Realm Studio application. Use this application to open the booking.realm file, and you can see data just submitted by the form.

Displaying all the stored data

Develop a new route handler to display all data records in our database:

/* Display all Bookings */
router.get('/bookings', function (req, res, next) {

  let bookings = bookingRealm.objects('Booking');

  res.render('bookings', {bookings: bookings});

});

Line 4 retrieves all the data and store it in the bookings variable. This variable is then passed to the bookings.ejs, a web template file which combines static HTML template with dynamic data.

Finally, develop this bookings.ejs in the views folder:

<!DOCTYPE html>
<html>
  <head>
    <title>All Bookings</title>
    <link rel='stylesheet' href='/stylesheets/style.css' />
  </head>
  <body>
    <table>
        <% bookings.forEach( function(booking) { %>
        
            <tr>
                <td><%= booking.name %></td>
                <td><%= booking.numTickets %></td>
            </tr>
        
        <% }); %>
     </table>
  </body>
</html>

Just a regular HTML file showing a table, together with some placeholders for the dynamic content. Those placeholders are specified by pairs of <%= and %>, which is a syntax of Embedded JavaScript (ejs).

Apart from settng up the placeholders, web template engine (ejs) also allows us to manipulate the dynamic content with a pair of <% and %>. Here, we loop over the bookings using the forEach() function. For each record we read, we name it as booking and display its details in the placeholders.

Visit http://localhost:3000/bookings to see all the inputted data.
